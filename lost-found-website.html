How This System Works
🔌
Hardware Layer
A Raspberry Pi Pico W runs MicroPython firmware. When the trigger button is pressed, the OV2640 camera captures a JPEG frame via SPI FIFO after VSYNC pulse. The onboard LED and buzzer give instant tactile feedback.
☁️
AI Identification
The JPEG is base64-encoded and uploaded to a FastAPI backend over campus WiFi. The backend calls Claude Vision API with a structured prompt that extracts object name, category, priority level, and a description useful for lost & found staff.
📋
Campus Registry
Every identified item is stored in SQLite with a unique LF-XXXX ID, timestamp, location, and image path. Staff can search, filter by priority, and mark items as claimed. Critical items like laptops and student IDs trigger urgent alerts.
